# RAG知识问答系统 - 高保真UI原型

## 项目概述

这是基于RAG（Retrieval-Augmented Generation）架构的知识问答系统高保真UI原型，包含完整的用户界面设计和交互逻辑演示。本原型采用现代化的Web技术栈，提供直观、美观且功能完整的用户体验。

## 🎯 项目特色

- **完整的业务流程覆盖**：涵盖从用户登录到知识问答、文档管理的全部功能
- **角色权限设计**：支持普通用户和管理员两种角色，功能权限清晰分离
- **响应式设计**：适配桌面和移动设备，确保在不同屏幕尺寸下的最佳显示效果
- **现代化UI设计**：采用Tailwind CSS构建，界面简洁美观，用户体验优秀
- **真实的交互逻辑**：JavaScript模拟真实业务场景，提供接近实际产品的操作体验

## 🚀 快速开始

### 环境要求

- Node.js 14.0 或更高版本
- 现代浏览器（Chrome 90+, Firefox 88+, Safari 14+, Edge 90+）

### 安装和启动

1. **克隆项目**（如果是从代码仓库获取）
   ```bash
   git clone <repository-url>
   cd ui
   ```

2. **安装依赖**
   ```bash
   npm install
   ```

3. **启动开发服务器**
   ```bash
   npm start
   ```
   
   或者使用开发模式（支持CORS）：
   ```bash
   npm run dev
   ```

4. **访问应用**
   
   打开浏览器访问：http://localhost:3000

### 替代启动方式

如果您没有安装Node.js，也可以使用以下方式之一：

#### 方式1: 使用Python（推荐用于开发测试）

```bash
# Python 3.x
python -m http.server 3000

# Python 2.x
python -m SimpleHTTPServer 3000
```

#### 方式2: 使用Live Server（VS Code插件）

1. 安装VS Code的Live Server插件
2. 右键点击`index.html`
3. 选择"Open with Live Server"

#### 方式3: 直接打开文件

直接双击`index.html`文件在浏览器中打开（某些功能可能受限）

## 💡 技术特性

### 前端技术栈
- **HTML5**：语义化标记，符合Web标准
- **Tailwind CSS**：原子化CSS框架，快速构建现代化界面
- **JavaScript (ES6+)**：现代JavaScript特性，模块化开发
- **FontAwesome**：丰富的图标库，提供专业的视觉体验

### 设计特点
- **响应式设计**：使用CSS Grid和Flexbox，适配各种屏幕尺寸
- **无障碍访问**：遵循WCAG指南，支持键盘导航和屏幕阅读器
- **加载性能**：CDN资源加载，优化首屏渲染时间
- **交互反馈**：丰富的动画效果和状态提示

### 模拟功能实现
- **用户认证**：模拟登录状态管理
- **数据交互**：Mock数据展示和CRUD操作
- **文件上传**：模拟文件处理流程和进度显示
- **实时通信**：模拟聊天对话和流式数据展示

## 🔧 自定义和扩展

### 修改样式
项目使用Tailwind CSS，您可以通过以下方式自定义样式：

1. **修改颜色主题**：在HTML文件中替换Tailwind的颜色类
2. **添加自定义CSS**：在各HTML文件的`<style>`标签中添加
3. **响应式调整**：使用Tailwind的响应式前缀（sm:, md:, lg:, xl:）

### 添加新页面
1. 创建新的HTML文件
2. 复制现有页面的基础结构
3. 在`index.html`的导航中添加链接
4. 更新其他页面的导航菜单

### 集成真实API
将模拟的JavaScript函数替换为真实的API调用：
```javascript
// 替换模拟函数
function mockApiCall() {
    // 模拟代码...
}

// 为真实API调用
async function realApiCall() {
    const response = await fetch('/api/endpoint');
    return response.json();
}
```

## 🐛 已知限制

由于这是UI原型项目，存在以下限制：

1. **数据持久化**：所有数据修改仅在当前会话中有效
2. **文件上传**：文件上传功能仅模拟流程，不会真实处理文件
3. **网络请求**：所有API调用都是模拟的，不会发送真实网络请求
4. **实时功能**：聊天和通知等实时功能通过setTimeout模拟
5. **浏览器兼容**：针对现代浏览器优化，IE等旧浏览器可能存在兼容性问题

## 🤝 开发建议

### 转换为生产应用

1. **后端集成**：将模拟的API调用替换为真实的后端接口

4. **数据验证**：添加表单验证和数据安全校验
5. **错误处理**：完善错误处理和用户反馈机制
6. **性能优化**：代码分割、懒加载、缓存策略等
